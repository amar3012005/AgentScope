# Evidence System: Source Attribution in BLAIQ Core

## Overview

The Evidence System ensures every answer generated by BLAIQ Core can be traced back to source documents. This builds trust and enables fact-checking.

## How Evidence Works

### 1. Source Registration

When documents are uploaded and processed:

```python
# Upload stage
POST /upload
{
  "files": ["report.pdf", "notes.txt"],
  "folder_path": "data/user01",
  "metadata": {
    "report.pdf": {
      "created_by": "John Doe",
      "department": "Engineering",
      "created_at": "2026-03-20T10:00:00Z"
    }
  }
}

# Each file gets a unique doc_id
doc_id = "report_" + hash(filename + folder_path)
# Example: "report_abc123def456"
```

### 2. Chunk Indexing with Attribution

During processing, chunks are created with full provenance:

```python
# Each chunk stores:
{
  "chunk_id": "report_abc123def456_chunk_001",
  "doc_id": "report_abc123def456",
  "original_filename": "report.pdf",
  "chunk_index": 0,
  "chunk_text": "Q1 revenue increased by 15% YoY...",
  "metadata": {
    "created_by": "John Doe",
    "department": "Engineering",
    "created_at": "2026-03-20T10:00:00Z"
  }
}
```

**In Qdrant payload:**
```json
{
  "doc_id": "report_abc123def456",
  "chunk_id": "report_abc123def456_chunk_001",
  "original_filename": "report.pdf",
  "text": "Q1 revenue increased by 15% YoY...",
  "chunk_index": 0,
  "metadata": {
    "created_by": "John Doe",
    "department": "Engineering",
    "created_at": "2026-03-20T10:00:00Z"
  }
}
```

### 3. Retrieval with Evidence Collection

When a query is answered, BLAIQ tracks which chunks contributed:

```python
# Query: "What was the Q1 revenue growth?"
retrieved_chunks = [
  {
    "chunk_id": "report_abc123def456_chunk_001",
    "doc_id": "report_abc123def456",
    "original_filename": "report.pdf",
    "text": "Q1 revenue increased by 15% YoY...",
    "retrieval_method": "vector_search",
    "similarity_score": 0.92,
    "ranking": 1,
    "metadata": {...}
  },
  {
    "chunk_id": "report_abc123def456_chunk_003",
    "doc_id": "report_abc123def456",
    "original_filename": "report.pdf",
    "text": "This growth was driven by...",
    "retrieval_method": "graph_search",
    "similarity_score": 0.85,
    "ranking": 2,
    "metadata": {...}
  }
]
```

### 4. Answer Generation with Evidence

LLM generates answer using retrieved chunks as context:

```python
system_prompt = """You are a helpful assistant. Provide accurate answers
based ONLY on the provided context. Always cite your sources."""

user_prompt = """Based on the context below, answer: What was Q1 revenue growth?

CONTEXT:
[Chunk 1] Q1 revenue increased by 15% YoY...
[Chunk 2] This growth was driven by...
[Additional chunks...]

Answer:"""

response = llm.generate(system_prompt, user_prompt)
# Response: "According to the Q1 report, revenue increased by 15% YoY..."
```

## Evidence Data Flow

```
Upload
  ↓
┌─────────────────────────────────┐
│ File Registration               │
│ • Generate doc_id               │
│ • Store metadata                │
│ • Create file index             │
└─────────────────────────────────┘
  ↓
Process
  ↓
┌─────────────────────────────────┐
│ Chunk Creation                  │
│ • Assign chunk_id               │
│ • Link to doc_id                │
│ • Preserve metadata             │
│ • Store in Qdrant payload       │
└─────────────────────────────────┘
  ↓
Query
  ↓
┌─────────────────────────────────┐
│ Retrieval                       │
│ • Collect chunks                │
│ • Track retrieval method        │
│ • Record similarity scores      │
└─────────────────────────────────┘
  ↓
Generate
  ↓
┌─────────────────────────────────┐
│ LLM Answer                      │
│ • Use chunks as context         │
│ • Generate natural response     │
│ • Preserve chunk references     │
└─────────────────────────────────┘
  ↓
Return
  ↓
┌─────────────────────────────────┐
│ Response with Evidence          │
│ • Answer text                   │
│ • Source documents              │
│ • Retrieved chunks              │
│ • Retrieval statistics          │
└─────────────────────────────────┘
```

## Evidence Use Cases

### 1. Fact Verification
```python
# User can verify any claim
response = retriever.query(
  "What was Q1 revenue?",
  include_evidence=True
)

# Click on answer → See original chunk
# Click on chunk → Download source PDF
# Check metadata → Verify timestamp, creator
```

### 2. Source Filtering
```python
# Query only documents from specific users/departments
response = retriever.query(
  "What are the budget constraints?",
  filter_metadata={
    "department": "Finance",
    "created_after": "2026-01-01"
  }
)
```

```python

```

## Best Practices for Evidence

### For Users
1. **Always review evidence** before relying on answers
2. **Check timestamps** to ensure document freshness
3. **Verify metadata** to confirm document source and creator
4. **Cross-reference** claims across multiple sources
5. **Download originals** for critical information

### For Operators
1. **Maintain metadata quality** - accurate doc creators and timestamps
2. **Monitor evidence usage** - track which documents are actually used
3. **Flag low-relevance sources** - remove documents with poor match scores
4. **Archive old documents** - maintain provenance over time
5. **Audit access logs** - track who queried what information

## Limitations & Future Work

### Current Limitations
- **Chunk-level only**: Evidence at chunk granularity, not paragraph level
- **No highlighting**: Cannot highlight exact cited text in PDFs
- **No temporal tracking**: Cannot show document version history
- **No access control**: Cannot restrict chunks by user permission level

### Future Enhancements
1. **Paragraph-level attribution**: Split chunks into sentences, track quotes
2. **PDF highlighting**: Return PDF with cited sections highlighted
3. **Version control**: Track document versions, maintain history
4. **Access control**: Filter chunks by user roles/permissions
5. **Confidence scoring**: Rate confidence in answer based on source quality
6. **Multi-hop evidence**: Show chain of reasoning across documents
