# TextBuddy — Formal Letter Artifact Skill

## Artifact Type
Formal letter — a structured business or official letter suitable for print or PDF.

## Template Structure

Generate the letter with these sections in order:

### 1. Sender Information Block
- Sender name, title, company name.
- Full mailing address (street, city, state/province, postal code, country).
- Phone number and email address.
- Pull from brand DNA or HITL answers. Omit fields not provided rather than using placeholders.

### 2. Date Line
- Full date format: "April 10, 2026" — never abbreviate.
- Place one blank line below the sender block.
- Use the date specified in HITL answers, or default to today's date.

### 3. Recipient Information Block
- Recipient name with appropriate title (Mr., Ms., Dr., etc.).
- Recipient title and company name.
- Full mailing address.
- Pull from HITL answers or evidence — do not fabricate recipient details.

### 4. Reference Line (Optional)
- Include only if a reference number, case ID, or subject is specified.
- Format as "Re: [subject]" on its own line, bold.

### 5. Salutation
- Use "Dear [Title] [Last Name]:" for formal letters.
- Use "Dear [First Name]," only when the relationship warrants familiarity.
- If the recipient is unknown, use "Dear Hiring Manager:" or "Dear Sir or Madam:" —
  never "To Whom It May Concern."

### 6. Opening Paragraph
- State the letter's purpose clearly in the first 1-2 sentences.
- Provide context: why you are writing and what outcome you seek.
- If this is a response, reference the prior communication with date and subject.

### 7. Body Paragraphs
- Develop the letter's argument or information in 1-3 focused paragraphs.
- Each paragraph addresses one topic or aspect of the purpose.
- Support claims with evidence findings — cite as `[source:ID]`.
- Use transitional phrases between paragraphs for logical flow.
- Maintain formal register throughout — no contractions, no colloquialisms.

### 8. Closing Paragraph
- Summarize the requested action or expected next step.
- Include a specific timeline if appropriate ("I look forward to your response by April 25").
- Express appreciation without excessive flattery.

### 9. Complimentary Close and Signature Block
- Formal: "Sincerely," or "Respectfully yours,"
- Professional: "Best regards," or "Kind regards,"
- Leave space for a handwritten signature (indicate with "[Signature]").
- Typed name, title, and company on separate lines below.

### 10. Enclosures/CC (Optional)
- "Encl: [document name]" if attachments are referenced.
- "cc: [name]" if copies are sent to additional recipients.
- Include only when specified in HITL answers or evidence.

## Formatting Rules

- Target one page (250-400 words for the body). Flag to the user if content exceeds one page.
- Use block format: all elements left-aligned, single-spaced with blank lines between sections.
- No indentation on paragraphs — use blank line separation instead.
- Standard business margins implied (1 inch / 2.54 cm all sides).

## Tone Rules

- Formal register throughout — write "cannot" not "can't," "will not" not "won't."
- Avoid passive voice unless convention requires it (legal, governmental contexts).
- Be direct but courteous — state requests clearly without being demanding.
- Match the level of formality to the letter's purpose:
  job application > client correspondence > vendor communication.
- Never use emoji, exclamation marks, or casual phrasing.

## Rules

- Never fabricate addresses, names, or titles — use only what evidence or HITL provides.
- If the letter's purpose is legal or regulatory, include a disclaimer that it does not
  constitute legal advice unless the sender is a licensed attorney.
- Maintain consistent spacing and structure — the letter must look professional when printed.
- If HITL answers specify a particular letter type (cover letter, complaint, recommendation),
  adapt the body structure to match that convention.
