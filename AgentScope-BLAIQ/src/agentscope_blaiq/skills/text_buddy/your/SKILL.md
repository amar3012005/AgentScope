# Perform a Competitive Market Analysis for Decentralized Compute Platforms

## Purpose
The purpose of this skill is to conduct a comprehensive competitive market analysis of decentralized compute platforms, with a specific focus on evaluating cost-per-token efficiency. This analysis will help identify the most cost-effective platforms, providing valuable insights for decision-making in selecting or investing in decentralized compute resources.

## When To Use
Use this skill when you need to:
- Compare different decentralized compute platforms based on their cost efficiency.
- Make informed decisions about which platform offers the best value for computational tasks.
- Understand the competitive landscape of decentralized compute platforms.
- Evaluate the economic feasibility of utilizing decentralized compute resources for specific projects or investments.

## Operating Instructions
1. **Identify Platforms**: Begin by listing all relevant decentralized compute platforms that are currently available in the market.
2. **Gather Data**: Collect data on each platform, focusing on the cost-per-token and any associated fees or costs that impact overall efficiency.
3. **Analyze Cost-Per-Token**: Calculate the cost-per-token for each platform, considering factors such as token price, transaction fees, and any additional costs.
4. **Compare Platforms**: Create a comparative analysis of the platforms based on the calculated cost-per-token efficiency.
5. **Report Findings**: Summarize the findings, highlighting which platforms offer the best cost efficiency and any notable trends or insights.

## Output Requirements
- A detailed report that includes:
  - A list of analyzed decentralized compute platforms.
  - The methodology used for calculating cost-per-token efficiency.
  - A comparative table or chart showing the cost-per-token for each platform.
  - A summary of key findings and recommendations.

## Constraints
- Ensure that the data used for analysis is up-to-date and sourced from reliable platforms.
- Focus solely on decentralized compute platforms; do not include centralized or hybrid solutions.
- The analysis should be objective and based on quantifiable data, avoiding subjective opinions.

## Examples

### Example 1
**Scenario**: A tech startup is considering using decentralized compute platforms for their data processing needs. They want to ensure they choose the most cost-effective option.

**Execution**:
- Identify platforms such as Golem, iExec, and Akash Network.
- Gather data on token prices, transaction fees, and any additional costs.
- Calculate and compare the cost-per-token for each platform.
- Provide a report recommending the platform with the lowest cost-per-token efficiency.

### Example 2
**Scenario**: An investor is evaluating potential investments in decentralized compute platforms and needs to understand which platform offers the best value.

**Execution**:
- Compile a list of potential investment platforms, including SONM and Ankr.
- Collect and analyze data on cost-per-token efficiency.
- Create a comparative analysis highlighting the most cost-efficient platforms.
- Deliver a report with insights into which platforms are likely to provide the best return on investment based on cost efficiency.
