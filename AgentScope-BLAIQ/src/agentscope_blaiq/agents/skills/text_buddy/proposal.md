# TextBuddy — Business Proposal Artifact Skill

## Artifact Type
Business proposal — a persuasive document presenting a solution, scope, timeline, and pricing
to a prospective client or internal stakeholder.

## Template Structure

Generate the proposal with these sections in order:

### 1. Executive Summary
- 3-5 sentences summarizing the opportunity, proposed solution, and expected outcome.
- Write this section last mentally but place it first in the document.
- State the core value proposition in the opening sentence.
- Include the total investment amount and timeline at a glance.
- This section must stand alone — assume the decision-maker reads only this.

### 2. Problem Statement / Opportunity
- Define the client's challenge or opportunity in 1-2 paragraphs.
- Use evidence findings to ground the problem with data — cite as `[source:ID]`.
- Mirror the client's language from HITL answers — show you understand their world.
- Quantify the cost of inaction if evidence supports it (lost revenue, inefficiency, risk).
- Do not editorialize — describe the problem factually.

### 3. Proposed Solution
- Describe the solution in 2-3 paragraphs with clear, specific deliverables.
- Explain how the solution addresses each aspect of the problem statement.
- Highlight differentiators — what makes this approach better than alternatives.
- Use evidence to back claims about the solution's effectiveness: case studies,
  benchmarks, or proven results cited with `[source:ID]`.
- Avoid technical jargon unless the audience is technical (adapt per HITL answers).

### 4. Scope and Deliverables
- List every deliverable as a numbered item with a clear description.
- Be precise: "Custom dashboard with 5 KPI widgets and PDF export"
  not "Analytics solution."
- Group deliverables by phase if the project has multiple stages.
- Explicitly state what is NOT included (out-of-scope) to prevent scope creep.
- Format as a table if there are 5+ deliverables:
  | # | Deliverable | Description | Phase |

### 5. Timeline / Milestones
- Present as a numbered list or table of milestones with dates or durations.
- Include key checkpoints: kickoff, reviews, delivery, sign-off.
- State dependencies and assumptions that could affect the timeline.
- If HITL answers specify a deadline, work backward from it.
- Format: **Phase 1: Discovery (Weeks 1-2)** — Description of activities.

### 6. Investment / Pricing
- Present pricing clearly — do not hide costs in dense paragraphs.
- Use a table for itemized pricing: | Item | Price |
- Show subtotals per phase if phased delivery.
- State the total investment prominently, bold.
- Include payment schedule: "50% upon signing, 50% upon delivery."
- Specify what triggers invoicing (milestone completion, calendar date, etc.).
- Currency formatting: symbol, thousands separator, two decimal places.

### 7. Terms and Conditions
- Keep to 3-5 essential terms unless HITL provides a full terms list.
- Standard items: payment terms, intellectual property, confidentiality,
  revision policy, termination clause.
- Write in plain language — avoid dense legalese.
- Include a validity period: "This proposal is valid for 30 days from the date above."
- Add a disclaimer that this is a commercial proposal, not a binding contract,
  unless HITL specifies otherwise.

### 8. Next Steps / Call to Action
- State exactly what the client should do to proceed.
- Provide a specific action: "Sign and return by [date]" or "Schedule a call at [link]."
- Include contact information for questions.
- Close with a confident, forward-looking sentence.

## Formatting Rules

- Use professional headings and consistent formatting throughout.
- Bold key figures: costs, dates, deliverable names.
- Use tables for structured data (pricing, timeline, deliverables).
- Target 2-5 pages depending on complexity. Adjust depth per HITL guidance.

## Tone Rules

- Confident but not arrogant — present the solution as proven, not as a favor.
- Client-focused: use "you/your" more than "we/our."
- Evidence-backed persuasion — let data make the case, not adjectives.
- Professional throughout — no exclamation marks, no hyperbole.

## Rules

- Never fabricate pricing, timelines, or deliverables — all must come from evidence or HITL.
- Every claim about results or capabilities must cite evidence.
- If critical sections lack evidence (e.g., no pricing info), note what is needed
  rather than inventing figures.
- Scope section must be specific enough to prevent disputes — vague deliverables are a liability.
- If HITL answers specify the client's industry, weave industry-specific language throughout.
